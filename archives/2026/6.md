## GitHub Actions 是什么？

**GitHub Actions 是 GitHub 内置的自动化工作流引擎，让你在代码仓库里直接编写 CI/CD 流水线。**

它解决的核心问题是：**「当我的仓库发生某件事时，自动执行一系列任务」**。

---

## 核心概念（面试必考）

| 概念 | 说明 | 类比 |
|------|------|------|
| **Workflow** | 一个完整的自动化流程，用 `.yml` 文件定义 | 工厂里的整条流水线 |
| **Event** | 触发 Workflow 的条件，如 `push`、`pull_request` | 按下启动按钮 |
| **Job** | Workflow 里的一个执行单元，可并行或串行 | 流水线上的一个工位 |
| **Step** | Job 里的单个任务，按顺序执行 | 工位上的一个操作 |
| **Action** | 可复用的最小命令单元，类似函数调用 | 预制的机械臂程序 |
| **Runner** | 执行 Job 的虚拟机/容器，GitHub 提供或自托管 | 流水线上的机器人 |

---

## 一个最简示例

```yaml
# .github/workflows/ci.yml
name: CI                          # 工作流名称

on:                               # 触发条件
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  test:                           # Job 名称
    runs-on: ubuntu-latest        # Runner 环境
    
    steps:
    - name: 拉取代码
      uses: actions/checkout@v4   # 复用官方 Action
    
    - name: 安装 Node.js
      uses: actions/setup-node@v4
      with:
        node-version: '20'
    
    - name: 安装依赖
      run: npm ci                 # 直接执行命令
    
    - name: 跑测试
      run: npm test
```

这个文件放在仓库的 `.github/workflows/` 目录下，push 代码后就会自动执行。

## GitHub Actions 的核心优势

### 1. 深度集成，无需额外工具
- 不需要 Jenkins 服务器，不需要 GitLab CI 配置
- 代码在哪，CI/CD 就在哪，**一个仓库搞定一切**

### 2. 海量 Action 生态（Marketplace）
> GitHub Marketplace 上有超过 2 万个现成的 Action

- `actions/checkout` — 拉代码
- `actions/setup-node/python/go` — 装环境
- `actions/cache` — 缓存依赖
- `actions/upload-artifact` — 保存构建产物
- `aws-actions/configure-aws-credentials` — 对接 AWS
- `docker/login-action` — 登录 Docker Hub

**你不需要写脚本，只需要「组装」别人写好的 Action。**

### 3. 矩阵构建（Matrix Build）
一次测试多个环境：

```yaml
strategy:
  matrix:
    node: [18, 20, 22]
    os: [ubuntu-latest, windows-latest]
    
steps:
  - run: npm test
```

GitHub 会自动生成 `2×3=6` 个并行任务，分别测试不同组合。

### 4. 强大的上下文和表达式
```yaml
env:
  COMMIT_SHA: ${{ github.sha }}
  IS_MAIN: ${{ github.ref == 'refs/heads/main' }}

steps:
  - if: github.event_name == 'pull_request'
    run: echo "这是PR触发的"
```

### 5. 自托管 Runner
解决网络、资源、成本问题：

```yaml
jobs:
  build:
    runs-on: self-hosted    # 用自己的机器跑
```

## 进阶能力（展示深度）

### 1. 作业依赖与并行控制
```yaml
jobs:
  build:
    runs-on: ubuntu-latest
  test:
    needs: build            # 等build完成后再跑
    runs-on: ubuntu-latest
  deploy:
    needs: [build, test]    # 等两者都完成
    if: github.ref == 'refs/heads/main'
    runs-on: ubuntu-latest
```

### 2. 环境与手动审批
```yaml
jobs:
  deploy-prod:
    runs-on: ubuntu-latest
    environment: production   # 需要审批
    steps:
      - run: deploy.sh
```

### 3. 复用工作流
```yaml
jobs:
  call-reusable:
    uses: .github/workflows/reusable.yml
    with:
      node-version: 20
    secrets:
      API_TOKEN: ${{ secrets.API_TOKEN }}
```

### 4. 缓存优化（面试高频）
```yaml
- uses: actions/cache@v3
  with:
    path: ~/.npm
    key: ${{ runner.os }}-node-${{ hashFiles('package-lock.json') }}
    restore-keys: |
      ${{ runner.os }}-node-
```
